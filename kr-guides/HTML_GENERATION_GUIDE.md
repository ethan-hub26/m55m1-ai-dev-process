# HTML Generation Guide

이 문서는 중국어 PDF를 한국어 HTML 가이드로 변환할 때 따라야 하는 고정 규격이다.  
다음부터 같은 작업은 이 문서를 작업 명세로 사용한다.

## 목적

- PDF 문서를 한국어 HTML 문서로 변환한다.
- 결과물은 개발자 전달용 정적 HTML 패키지여야 한다.
- 브라우저에서 바로 열 수 있어야 하며 별도 빌드가 없어야 한다.

## 산출물 구조

- 기준 루트: `H:\ebk\Bsp\deliverables\kr-guides\`
- 문서별 폴더: `deliverables/kr-guides/<doc-slug>/`
- HTML 파일: 문서별 폴더 바로 아래 1개
- 이미지 폴더: `assets/<short-symbolic-name>/`
- 페이지별 추출 폴더: `page-source/`
- 생성 스크립트: `H:\ebk\Bsp\scripts\generate_<doc-slug>.py`

예시:

- `deliverables/kr-guides/person-track-kr/Person_Tracking_KR_Guide.html`
- `deliverables/kr-guides/person-track-kr/assets/tracker/`
- `deliverables/kr-guides/person-track-kr/page-source/pages/page-03.md`

## 폴더 이름 규칙

- 문서별 폴더명은 짧고 일관된 slug를 사용한다.
- 이미지 하위 폴더명도 짧고 상징적인 이름을 사용한다.
- 문서마다 독립 패키지처럼 관리한다.

## HTML 스타일 규칙

- 기준 레이아웃과 스타일은 다음 참고 페이지를 따른다.
- `https://ethan-hub26.github.io/UM_Nuvoton_Nu-Link_Debug_Adapter_KR_Guide.html`
- 상단 헤더, 메타 칩, 좌측 목차, 우측 본문 카드형 섹션 구조를 유지한다.
- 목차는 깔끔해야 하며 과도하게 세분화하지 않는다.
- 반응형으로 동작해야 한다.
- 단일 HTML 내 `<style>` 블록 방식으로 유지한다.
- 외부 JS는 사용하지 않는다.

## 본문 구성 규칙

- 단순 요약본이 아니라 문서 내용을 빠짐없이 채운다.
- 결과 화면만 남기지 말고 앞단의 흐름도 모두 포함한다.
- HTML 작성 전에 페이지별 추출본을 먼저 만든다.
- 페이지별 추출본을 한 페이지씩 대조한 뒤에만 HTML 본문에 반영한다.
- 페이지별 추출본 없이 바로 HTML부터 쓰는 방식은 금지한다.
- 반드시 다음 성격의 내용을 채운다.
  - 프로젝트 개요
  - 사용 시나리오
  - 데이터셋 준비
  - 개발 환경 구축
  - 학습 과정
  - 모델 변환 과정
  - 보드 추론 과정
  - 후처리와 표시 과정
  - DEMO 링크
- 문서 구조는 읽기 좋은 문서형으로 재구성하되, 내용 누락은 금지한다.
- 이미지 설명은 해당 이미지 바로 위에 둔다.
- 위에서 설명을 몰아서 하고 아래에 이미지만 나열하는 구성은 금지한다.

## 번역 규칙

- 중국어는 최종 HTML에서 보이지 않게 한다.
- 제목, 본문, 표, 캡션, 설명은 한국어로 작성한다.
- 제품명, 툴명, 파일명, 인터페이스명, 모델명은 원어 영문 표기를 유지해도 된다.
- 중국어 병기는 기본적으로 하지 않는다.
- 한국어로 번역되었다면 중국어 대목차나 중국어 표기를 함께 넣지 않는다.
- 링크 URL은 그대로 유지해도 된다.
- 링크 라벨은 한국어로 이해 가능하게 적는다.

## 금지 표현

- `슬라이드 3`, `슬라이드 13-18` 같은 표기 금지
- `슬라이드 대응`, `슬라이드 순서` 같은 표현 금지
- ``, `PDF의`, `화면을 유지하되` 같은 서술 최소화
- `복구했습니다`, `잘라서`, `분리했습니다`, `넓게 추출했습니다`처럼 편집 과정을 설명하는 문구 금지
- 이미지 설명은 편집 방식이 아니라 이미지 내용만 설명한다.

## 이미지 처리 규칙

- 페이지 전체를 통째로 캡처하지 않는다.
- 필요한 이미지, 사진, 코드, UI 영역만 잘라 사용한다.
- 이미지 안에 중국어가 남으면 그대로 두지 않는다.
- 우선순위는 다음과 같다.
  - 중국어가 없는 필요한 영역만 다시 캡처
  - 한국어 설명으로 대체 가능한 경우 HTML 본문/표/리스트로 대체
  - 코드라면 이미지보다 코드 블록 또는 설명 카드로 전환 가능
- 사용하지 않는 전체 페이지 캡처 이미지는 최종 산출물에서 제거한다.
- 다만 추후 재생성용 스크립트는 삭제하지 않는다.

## 캡션/설명 규칙

- 이미지 바로 위 `fig-note`는 한 문장 설명만 사용한다.
- 예시:
  - `Kaggle 데이터셋 다운로드 페이지입니다.`
  - `InsertTopNDetection 함수 구현입니다.`
  - `DetectionResult 구조 정의입니다.`
- 피해야 할 예시:
  - `슬라이드 22는 다운로드 화면이 중심이라 필요한 영역만 분리했습니다.`
  - `슬라이드 42의 코드 영역을 복구했습니다.`

## 링크 처리 규칙

- URL은 값을 유지한다.
- 설명 라벨은 한국어 중심으로 적는다.
- `Link` 같은 라벨은 가능하면 한국어 설명으로 바꾼다.

## 검수 기준

- `page-source/pages/page-xx.md` 파일이 페이지별로 존재해야 한다.
- `page-source/index.md`와 `page-source/manifest.json`이 있어야 한다.
- 페이지별 추출본과 HTML을 대조하면서 누락 여부를 확인해야 한다.
- HTML에 중국어가 남아 있지 않아야 한다.
- HTML에 `슬라이드` 표기가 남아 있지 않아야 한다.
- HTML에 편집 과정 설명 문구가 남아 있지 않아야 한다.
- 모든 `<img>`는 실제 파일을 참조해야 한다.
- 목차 앵커는 모두 동작해야 한다.
- 문서 초반부터 후반까지 내용 누락이 없어야 한다.
- 학습/변환/보드 추론 같은 핵심 개발 흐름이 빠지면 실패다.
- 이미지가 필요한데 빠져 있으면 실패다.

## 작업 순서

1. PDF 구조와 페이지 수를 확인한다.
2. 산출물 폴더와 이미지 폴더를 문서별로 만든다.
3. `page-source/`를 만들고 페이지별 추출본을 생성한다.
4. 텍스트와 필요한 이미지 영역을 추출한다.
5. 페이지별 추출본을 한 페이지씩 대조하면서 중국어를 한국어 본문으로 재구성한다.
6. 참고 HTML 스타일에 맞춰 정적 HTML을 작성한다.
7. 이미지 설명을 각 이미지 바로 위에 붙인다.
8. 중국어, `슬라이드`, 편집 메타 문구를 제거한다.
9. 이미지 참조, 앵커, 내용 누락 여부를 검수한다.

## 페이지별 추출본 생성 명령

페이지별 추출본을 먼저 생성하고, 이 파일을 한 페이지씩 확인한 뒤 HTML을 작성한다.

예시:

```powershell
python H:\ebk\Bsp\scripts\export_pdf_page_sources.py `
  --pdf-module scripts.generate_person_tracking_kr_cropped `
  --assets-module scripts.generate_person_tracking_kr_cropped `
  --out H:\ebk\Bsp\deliverables\kr-guides\person-track-kr\page-source `
  --page-start 3 `
  --page-end 47
```

생성 결과:

- `page-source/index.md`
- `page-source/manifest.json`
- `page-source/pages/page-xx.md`

이 파일들을 기준으로 페이지별 누락 여부를 먼저 점검하고, 그 다음 HTML을 수정한다.

- `이 PDF를 같은 기준으로 HTML로 변환`

- `이미지는 필요한 영역만 사용`
- `개발환경 구축, 학습, 모델 변환, 보드 추론까지 모두 채우기`

## 기본 명령 템플릿

다음 요청은 아래 의미로 처리한다.

`<문서명>.pdf를 기존 KR guide 규격대로 변환해줘`

내부 수행 기준:

- `deliverables/kr-guides/<slug>/`에 독립 패키지 생성
- 한국어 HTML 1개 생성
- `assets/<short-name>/`에 필요한 이미지 자산만 저장
- `page-source/`에 페이지별 추출본 생성
- 중국어 제거
- 슬라이드 번호 제거
- 편집 과정 설명 제거
- 설명은 이미지 바로 위에 배치
- 개발 절차와 결과까지 전체 내용 반영
